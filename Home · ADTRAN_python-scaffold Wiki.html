Getting Started

Clone repository and Replace Template Values

# <new_repository_name> shoud use github naming conventions hyphen (-) for separators
git clone git@github.adtran.com:ADTRAN/python-scaffold.git <new_repository_name>
rename.sh <new_package_name>
rm -rf rename.sh .git

Run tests

You can run either tox or make.

The Makefile provides wrappers for each tox environment in tox -l. It also provides docker wrappers for each environment. See the output of make usage for more details.

Pushing to GitHub

After being granted permissions to commit to your new GitHub repository, run the following.

git init
git add -A
git commit -a -m "Initial commit of <new_package_name>"
git remote add origin git@github.adtran.com:ADTRAN/<new_package_name>
git push -u origin master

This will automatically create a jenkins pipeline job for your project. Job creation may not be immediate. You can expect at most one day of delay. If you need your job sooner, then you can use the Run Now option for polling. The badge at the top of the README will link directly to the job once it is created.

By default, the Jenkinsfile runs analysis and tests against python 2.7 and python 3. If you don't want automatic job creation you can remove the Jenkinsfile. If you don't plan to support python 3, you can remove py3 references from the Jenkinsfile and from tox.ini.

Tox Overview and Template Configuration

Tox is a tool designed specifically for python to remove common boilerplate from the development workflow.

This below environments will be ran if "tox" is ran without any arguments.

$ tox -l
analysis
py27-test
py3-test

Note that the commands listed by "tox -l" are not the only environments that can be ran. Any environment listed under the "commands" section of tox.ini are valid environments. These may be ran using "tox -e environment_name". For example, if using the template, "tox -e py27-unit" will run tests and generate coverage data for the tests under test/unit for python 2.7.

Requirements for Using Included tox Sample
